mpvue开发小程序踩坑记录

2019-04-07 20:16:42

本文用来记录mpvue开发小程序踩坑历程。

# 介绍

    最近在做一个小程序，在Github上看了一下发现美团出了一个mpvue —— vue版的小程序（其实是vue转换为原生小程序代码），果断利用起来，mpvue开发小程序与原生开发vue在一些地方有着不同的写法，这里来做一下记录。

# 正文

# Input键盘回车事件

在vue中，输入框的键盘回车事件是这样的：

<!-- 省略其他代码 -->
<input @keyup.native.enter="handleSubmit" />

但是在mpvue中，不管是去掉native还是如何如何都无法触发enter事件，在网上搜了一下才发现原来是这样写的；

<!-- 省略其他代码 -->
<input @confirm="handleSubmit" />
<!-- 转换后的代码 -->
<input bindconfirm="handleProxy" />

# 标签转换

<i></i>标签会被转换为<view></view>标签，
<span></span>则转换为<label></label>标签。

<!-- i标签 -->
<i class="iconfont"></i>
<!-- 转换后变为块级元素 -->
<view class="_i iconfont"></view>
<!-- span -->
<span class="icon"></span>
<!-- 转换后依旧是行内元素 -->
<label class="_span iconfont"></label>

对于习惯使用<i></i>标签做图标的话当然是希望图标是行内元素了，但是突然转换为块级元素就需要再去更改样式，如此只能被迫改为span标签了。

# 获取页面url参数

对于不支持路由的小程序当然是和正常vue的写法有区别了。

如获取`http://localhost:8080/bolg/id`的id值
// vue写法
const id = this.$route.params.id;
// mpvue小程序写法
const id = this.$root.$mp.query.id

# axios配置

在mpvue中直接使用axios会报错，原因是小程序和浏览器的环境不同，wx有自己的请求方法（wx.request），而不是xhr。

这时我们需要对axios做一些修改，在node_modules/axios/dist/axios.js中

// 注释掉这段代码
    axios.all = function all(promises) {
      return Promise.all(promises);
    };

// 添加mpvue对axios的适配器,实际上是对wx自带请求方法的封装
      axios.defaults.adapter = function (config) {
        return new Promise((resolve, reject) => {
          //  console.log(config)
      let data = config.method === 'get' ? config.params : config.data
          // wx小程序发起请求
          wx.request({
            url: config.url,
            method: config.method,
            data: data,
            success: (res) => {
              return resolve(res)
            },
            fail: (err) => {
              return reject(err)
            }
          })
        })
      }

WARNING

说明：axios的GET请求方式要使用params方式(主要是为了对应适配器，其实也可以在适配器中通过url提取出get的参数，不过相对的要更麻烦)

  axios.get("http://localhost:3000/get_msg", {
      params: {
        ID: 12345
      }
    }).then(res => {
      console.log(res);
    }).catch(err => {
      console.log(err);
    });

    可能写的比较抽象，这个需要自己动手配置一下才比较好理解，可以参考我写的demo——mpvue中适配axios

# 合法域名校验问题

调试的时候如果报非法域名错误，需要勾选详情中的不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书

# 总结

    对于mpvue，可以说是对vue开发人员非常的友好，相比同期的wepy（同vue转，但不支持html标签和vuex）,taro（react转），个人觉得mpvue真的是不要太友好啊，尤其是通过vuex进行数据管理而不用去深究小程序代码大大提高了开发效率。
