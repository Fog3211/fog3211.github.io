聊一聊JavaScript中的事件绑定、事件监听和事件委托

2019-03-31 21:55:24

本文将介绍js中的事件绑定，事件监听和事件委托。

# 介绍

   做前端的对js事件肯定都不陌生，在js中，如果要想触发事件，那就必须先进行事件函数与dom节点的绑定，通过触发条件进行事件的处理。下面我就来介绍一下js中的事件绑定。

# 正文

# 事件绑定

这个应该是最基础的，常见的有两种方法，如下：

<ul>
    <li onclick="hello()">item1</li>
    <li id="item2">item2</li>
</ul>
<script>
// DOM中直接绑定事件
    function hello() {
        alert("hello");
    }
// js中绑定事件
let item2 = document.getElementById("item2");
    item2.onclick = function () {
        alert("hello item2");
    }
</script>

# 事件监听

# 绑定多个事件

主要利用的是addEventListener，通过该函数我们可以进行节点操作的监听工作。 比如需求如下：点击item打印其对应的值。

<ul>
    <li class="item">item1</li>
    <li class="item">item2</li>
    <li class="item">item3</li>
    <li class="item">item4</li>
    <li class="item">item5</li>
</ul>

let item_list = document.getElementsByClassName("item");
    for (let i = 0; i < item_list.length; i++) {
        item_list[i].addEventListener("click", function () {
            console.log(this.innerHTML);
        });
    }

看起来和事件绑定差不多，但是有一点，就是事件绑定只会执行最后一次绑定的事件，而事件监听可以绑定多个事件。

<!-- 事件绑定  只会显示hello2 -->
<button id="btn">button</button>
<script>
    let btn = document.getElementById("btn");
    btn.onclick = function () {
        alert("hello1");
    }
    btn.onclick = function () {
        alert("hello2");
    }
</script>
<!-- 事件监听 可以显示hello1，hello2 -->
<button id="btn">button</button>
<script>
    let btn = document.getElementById("btn");
    btn.addEventListener("click", hello1);
    btn.addEventListener("click", hello2);

    function hello1() {
        alert("hello1");
    }

    function hello2() {
        alert("hello2");
    }
</script>

# 冒泡or捕获

这只是事件监听的一个功能，它还有另一个强大的功能就是可以设置事件的执行阶段在冒泡阶段还是捕获阶段。

//三个参数分别对应触发条件，需要执行的函数，在什么阶段执行（可省略，默认冒泡）
addEventListener(event,function,useCapture)

下面通过代码来解释冒泡和捕获的区别。

<div id="container">
    <button id="btn">button</button>
</div>
<script>
    let container = document.getElementById("container");
    let btn = document.getElementById("btn");

    function hello1() {
        alert("hello1");
    }

    function hello2() {
        alert("hello2");
    }
    // 输出结果和定义顺序无任何关系，可以自己试一下
    // 输出hello1，hello2
    container.addEventListener("click", hello1, true);
    btn.addEventListener("click", hello2, true);
    // 输出hello2，hello1
    container.addEventListener("click", hello1, false);
    btn.addEventListener("click", hello2, false);
</script>

通过以上实验就可以看出，冒泡过程其实是从事件触发的节点逐渐向上执行，即btn -> div -> body -> document ->window，而捕获阶段则是相反。

# 解除绑定

对于事件绑定我们知道的一旦绑定上事件就不能解绑了，但是对于事件监听而言，是可以进行事件解绑的（其实是取消监听）。


// 与addEventListener对应，没什么好说的
removeEventListener(event,function,useCapture);

主要用途：说实话在项目中其实并不常见，可能在执行了几次事件之后需要解绑什么的，主要看你的需求有多奇葩了。

# 事件委托

事件委托就是利用冒泡的原理，把事件加到父元素或祖先元素上，触发执行效果。

# 优化代码结构

<ul id="list">
    <li id="item1">item1</li>
    <li id="item2">item2</li>
    <li id="item3">item3</li>
</ul>

let item1 = document.getElementById("item1");
let item2 = document.getElementById("item2");
let item3 = document.getElementById("item3");

// 事件绑定
item1.onclick = function () {
    alert("hello item1");
}
item2.onclick = function () {
    alert("hello item2");
}
item3.onclick = function () {
    alert("hello item3");
}

// 事件委托
document.addEventListener("click", function (event) {
    let target = event.target;
    if (target == item1) {
        alert("hello item1");
    } else if (target == item2) {
        alert("hello item2");
    } else if (target == item3) {
        alert("hello item3");
    }
})

可以看出如果li标签成几何倍数增加的话，通过事件绑定的方式会绑定每一个li节点，就会严重降低JavaScript的性能。而事件委托的话，是可以通过if语句的判断来并进行唯一一个节点的绑定，对比来看，事件委托的优点十分明显。

事件委托对性能的提升方面可以看一下这篇文章。

# 动态增加节点

<ul id="list">
    <li id="item1">item1</li>
    <li id="item2">item2</li>
    <li id="item3">item3</li>
</ul>

 let list = document.getElementById("list");

// 事件绑定方法不会触发item4
let item = list.getElementsByTagName("li");
for (let i = 0; i < item.length; i++) {
    item[i].onclick = function () {
        alert(item[i].innerHTML);
    }
}
// 事件委托方法可以触发item4
document.addEventListener("click", function (event) {
    let target = event.target;
    if (target.nodeName == "LI") {
        alert(target.innerHTML);
    }
})

let node = document.createElement("li");
let textnode = document.createTextNode("item4");
node.appendChild(textnode);
list.appendChild(node);

在DOM中动态的添加节点，不会自动添加事件的绑定（其实放在节点插入的下面是可以的，但总不能一直这样做吧），事件委托可以自动检测新的DOM事件，因为它是通过监听来进行事件的绑定的。

# 总结

    对于三种事件与DOM结合的方式，用的最多的应该还是事件绑定，这个确实比较简单方便。事件监听的话功能更为强大，处理冒泡，捕获阶段的事件比较好用。事件委托算是高级一点的了，有利于代码结构的优化及性能的提升。
