刷新页面保存Vuex状态

2019-10-23 16:05:55

本文将介绍如何解决刷新页面Vuex丢失的问题。

# 介绍

    用过vue的同学应该都知道组件间传值的方式大概可以分为父传子（props）,子传父（$emit），或者无直接联系的组件（eventbus）,或者使用官方推荐的状态管理工具——Vuex，然而这里有一个问题，那就是当刷新页面时，vuex会被清空（因为是存储在当前内存中的），下面就来讲讲问题的解决方法。

# 正文

# 思路

    对于上面的问题，不依赖外部存储是不太好解决的啦，因为刷新页面必然会导致当前内存的释放，因此我们需要将vuex中的数据保存到其他地方，那保存到哪里好呢？（后端数据库，sessionStorage，localStorage，cookie）。

    鉴于vue一般适用于单页面应用，因此选择sessionStorage是最合适不过的了(只要当前页面不关闭就一直有效)，对其他的来讲：

后端数据库:每次刷新页面还要多发一次请求，对性能而言并不好.

localStorage:存储到本地，但是vuex一般用于存储登录状态之类的东西，放在这里相对sessionStorage肯定是差一点的了.

cookie:由于大小限制（4K），不可能作为存储的最优选择.

# 实现

export default {
  created () {
    //在页面创建/刷新时从sessionStorage中获取数据
    if (sessionStorage.getItem("store") ) {
        this.$store.replaceState(Object.assign({}, JSON.parse(sessionStorage.getItem("store"))))
    }

    //在页面刷新前将vuex里的信息保存到sessionStorage里
    window.addEventListener("beforeunload",()=>{
        sessionStorage.setItem("store",JSON.stringify(this.$store.state))
    })
  }
}

# 思考

    其实vuex里面是状态放在sessionStorage也是一样的，这样就省去了再存储的过程，但是为什么不这样做呢？

当然是因为vuex里面的数据是响应式的，如果失去了响应式那vuex的优势就不存在了（总不能为了监听sessionStorage里面的数据在每一个页面中都加入watch属性吧）。

# 总结

    其实对于该问题的解决方法大同小异，基本都是在刷新时靠外部存储保存来实现的，当然也有许多封装好的功能更强大的包可以直接拿来用，这里就只是抛砖引玉啦。
