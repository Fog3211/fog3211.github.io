详解js基本类型——Symbol

2019-04-28 21:34:32

本文将介绍js中的基本类型——Symbol。

# 介绍

    在ES5的对象中，如果对象的属性名相同，则会出现错误，因为对象的键只能是唯一的。为了解决这个问题，ES6就引入了Symbol。

//这时会报错，因为对象中有两个属性的名相同
var obj={
  a:1,
  b:2,
  a::3
}
console.log(obj);

# 正文

# 概念

    Symbol是ES6新引入的第6个原始数据类型（之前的null，undefined，String，Boolen，Object），主要用于表示独一无二的值，相当于id的作用。

# 使用

# 创建

// 基本用法
let s = Symbol();

console.log(typeof s);// "symbol"

加入字符串参数更容易区分

// 带字符串参数
let s1 = Symbol('aaa');
let s2 = Symbol('bbb');

console.log(s1) // Symbol(aaa)
console.log(s2) // Symbol(bbb)

WARNING

Symbol函数前不能使用new命令，否则会报错。这是因为生成的 Symbol 是一个原始类型的值，不是对象。

# 比较

因为Symbol是独一无二的，因此就算字符串参数相同的Symbol也是不一样的

// Symbol()也是一样的道理，相当于参数为空
let s1 = Symbol('aaa');
let s2 = Symbol('aaa');

console.log(s1===s2) // flase

Symbol 值不能与其他类型的值进行运算，否则会报错

// 错误例子
let s1 = Symbol('aaa');
let s2 = Symbol('bbb');

console.log(2*s1);
console.log('B'+s2);

# 作为属性名

var s = Symbol();

// 第一种写法
var obj = {};
obj[s] = 'Hello';

// 第二种写法
var obj = {
  [s]: 'Hello'
};

// 第三种写法
var obj = {};
Object.defineProperty(obj, s, { value: 'Hello' });

console.log(obj[s]); // "Hello!"

WARNING

symbol做属性名时，不能用.运算符,如obj.s

# Symbol.for()，Symbol.keyFor()

虽然symbol可以创造出独一无二的值，但有时我们可能需要使用重复的symbol，这时就可以用Symbol.for()来实现了。

let s1 = Symbol.for('aaa');
let s2 = Symbol.for('bbb');

s1 === s2 // true

通过Symbol.keyFor()可以获取到设置的值。

let s1 = Symbol.for("aaa");
Symbol.keyFor(s1) // "aaa"

let s2 = Symbol("bbb");
Symbol.keyFor(s2) // undefined

    这里只是简单介绍了基本的用法（其他的一般用不到），想要其他深入用法可以移步阮一峰老师的文章ECMAScript 6 入门

# 总结

    作为ES6新增的一种基本类型，其实能用到的地方还是比较有限的，唯一的值完全可以通过id来代替，个人感觉symbol主要的用途还是在那些需要唯一键但又不关心它是什么的时候（如vue或react的for循环遍历中的key值）。
