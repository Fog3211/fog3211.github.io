前端必备技能之跨域篇

2019-01-20 21:06:55

本文将介绍几种前端跨域方式。

# 介绍

# 什么是跨域

对于前端小伙伴们来说，跨域这个词应该并不算陌生，在做项目中与后端的对接会经常遇到，为了弄清楚跨域，就需要先了解什么是同源。先看一下MDN的介绍。

TIP

如果两个页面的协议，端口（如果有指定）和域名都相同，则两个页面具有相同的源。

也就是符合协议，域名，端口三者都相同的即为同源。

# 为什么会产生跨域

TIP

同源策略限制了从同一个源加载的文档或脚本如何与来自另一个源的资源进行交互。这是一个用于隔离潜在恶意文件的重要安全机制。

简单讲就是基于安全因素的考虑，浏览器一般会禁止跨域访问，保证浏览器不易受到XSS、CSFR等攻击。不允许跨域访问并非是浏览器限制了发起跨站请求，而是跨站请求可以正常发起，但是返回结果被浏览器拦截了。

# 跨域解决方法

  1. 跨域资源共享（CORS）
  2. JSONP跨域
  3. nginx代理跨域
  4. location.hash跨域
  5. window.name跨域
  6. HTML5的postMessage跨域
  7. document.domain跨域
  8. WebSocket协议跨域
  9. 修改浏览器的跨域设置

这里主要介绍下前三种，也是目前比较流行的解决方式。

# 准备测试demo

    先写一个简单的测试代码（点击按钮就会发起ajax请求，并将返回信息打印在控制台）。
前端ajax请求

<body>
    <div>
        <input type="text" placeholder="未收到数据" id="showMessage">
        <button id="sendBtn">获取数据</button>
    </div>
</body>
<script>
    var showMessage = document.getElementById("showMessage");
    var sendBtn = document.getElementById("sendBtn");
    sendBtn.onclick = function () {
        var xhr = new XMLHttpRequest();
        xhr.open('get', 'http://localhost:3000/getData');
        xhr.send();
        xhr.onreadystatechange = function () {
            if (xhr.readyState == 4 && xhr.status == 200) {
                showMessage.value = xhr.responseText;
                console.log(xhr.responseText);
            }
        }
    }
</script>

后端express服务器

var express = require("express");
var app = express();
const port = 3000;

app.get("/getData", (req, res) => {
  res.send("已获取到后端数据了！");
});

app.listen(port, function () {
  console.log("server is on port " + port);
})

# 解决方案

# CORS

先看一下MDN的介绍

TIP

跨域资源共享(CORS) 是一种机制，它使用额外的 HTTP 头来告诉浏览器 让运行在一个 origin (domain) 上的Web应用被准许访问来自不同源服务器上的指定的资源。当一个资源从与该资源本身所在的服务器不同的域、协议或端口请求一个资源时，资源会发起一个跨域 HTTP 请求。

解决：在后端中加入以下代码即可

app.all("*", function (req, res, next) {
  res.header("Access-Control-Allow-Origin", "*");
  res.header("Access-Control-Allow-Credentials", true);
  res.header("Access-Control-Allow-Headers", "Content-type");
  res.header("Access-Control-Allow-Methods", "PUT,POST,GET,DELETE,OPTIONS,PATCH");
  res.header("Access-Control-Max-Age", 1728000);
  next();
});

需要深入了解的请移步跨域资源共享 CORS 详解——阮一峰

# JSONP

JSONP，全称为JSON with Padding，是json的一种补充使用方式，利用script标签中的src可以跨域访问（比如引用jQuery的CDN）来解决跨域问题。JSONP非官方协议，它只是前后端一个约定，如果请求参数带有约定的参数，则后台返回javascript代码而非json数据，返回代码是函数调用形式，函数名即约定值，函数参数即要返回的数据。

前端callback回调即可

 var showMessage = document.getElementById("showMessage");
   var sendBtn = document.getElementById("sendBtn");
   sendBtn.onclick = function () {
       var script = document.createElement("script");
       script.src = "http://127.0.0.1:3000/jsonp?callback=handleResponse";
       document.body.insertBefore(script, document.body.firstChild);  
   }
   function handleResponse(response){
           console.log(response);
           showMessage.value=response.data;
   }

后端

app.get('/jsonp',function(req,res,next){
  res.jsonp({data:'已通过jsonp获取到后端数据了！'});
  });

WARNING

JSONP只对GET请求方式有效。

# nginx

首先用nginx搭建静态服务器，配置如下

 listen       8080;
        server_name  localhost;

        #charset koi8-r;

        #access_log  logs/host.access.log  main;

        location / {
            root   cross-domain-test/; #此处为index页面项目目录
            index  index.html index.htm;
        }

nginx反向代理配置

#nginx.conf中添加  
server{
    listen 8080;
    server_name localhost;
    #凡是localhost:8080/apis这个样子的，都转发到真正的服务端地址http://localhost:3000
    location ^~ /apis {
        proxy_pass http://localhost:3000;
    }  
}

    原理：nginx通过反向代理将访问localhost:8080/apis的请求转发到http://localhost:3000上，因此将前端ajax请求地址改为相同域名相同端口即可。

  xhr.open('get', '/apis/getData');

# 总结

对于CORS，JSONP，nginx三者的使用场景：

CORS配置简单方便，但不支持较低版本的浏览器。

nginx不需要后端协助，但配置过程相较其他两个较为麻烦，适用于大型项目。

JSONP只能用于GET请求方法，但支持较老版本的浏览器。
