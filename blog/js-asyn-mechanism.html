Javascript单线程和异步机制

2019-06-26 20:42:05

本文将介绍Javascript单线程和异步机制。

# 介绍

    众所周知，JavaScript是一个单线程的语言，那么这就意味着它在处理复杂问题的时候必须要花费更多时间，好在js里面有个叫异步的东西，可以很好的解决这个问题。

# 正文

# 同步

    简单理解就是类似队列一样。后面的事项需要等待前面的事项执行结束后再去执行。如：

var aa = 1+2; //第一步
console.log(aa); //第二步

# 异步

    异步与同步相对应，有点类似平行线，通过其他线程来执行另外的任务，等任务执行完毕后才在回到主线程。说到这，可能扯到了js的线程问题：

# js线程与浏览器线程

    JavaScript是单线程的，这在它出生的时候就已经定了，那为什么还能分主次线程呢？其实是因为负责执行其他任务的是浏览器线程（浏览器是多线程的），js线程负责任务队列。其他线程（GUI渲染线程，事件触发线程，定时触发器线程，异步http请求线程）用于辅助主线程（JS引擎线程）。

# 执行顺序

在异步中，首先执行js线程里的任务，当主线程任务执行完成之后再去执行其他线程处理好又重新放到主线程队列里的任务。如：

// js主线程任务优于其他线程任务（定时器任务）执行，即使定时器为0，顺序在前。
setTimeout(function() {
    console.log("a")
}, 0);

console.log("b");
// 结果：b  a

# 回调函数

    对于类似ajax请求的任务，当js完成ajax.send()操作后，js主线程不需要管接下来的事情了，只需要执行当前队列的下一个任务，知道ajax请求完成再去把回调函数放入队列里面依次执行即可。

# 总结

    所谓js的异步机制，其实就是通过浏览器的其他线程对js主线程的辅助，通过回调函数将其他线程的处理结果返回给js主线程，从而避免拥堵，减少不必要的等待时间。
