使用nginx在同一域名下部署多个项目

2019-05-01 22:59:47

本文将介绍nginx如何在同一域名下部署多个项目。

# 介绍

    前几天部署项目的时候，因为项目文件打包比较大，直接部署到github上的话会比较慢，于是就想部署到自己的云主机上，在开启gzip来进行性能优化。
    出于对未来可能会部署多个项目的考虑，就学习了下使用nginx在同一域名下部署多项目。

# 正文

    主要有两种方式：一个是在域名下建立不同的子域名进行部署，另一个是通过域名加目录来进行部署。以下以xxx.com域名，项目aaa，项目bbb，ip地址123.123.123.123为例。

# 子域名

首先，先在域名服务商处解析出二级域名，添加A记录即可。

然后在nginx.conf中配置即可，这里监听的都是80端口，只是子域名和项目目录不同。

#主机名对应项目目录即可
server {
    listen       80;
    server_name  aaa.xxx.com;

    location / {
        root   projects/aaa/;         #项目aaa目录
        index  index.html index.htm;
    }
}

server {
    listen       80;
    server_name  bbb.xxx.com;

    location / {
        root   projects/bbb/;         #项目bbb目录
        index  index.html index.htm;
    }
}

# 域名+目录

    上面的方法相对简单，但我们可能更希望通过xxx.com/aaa,xxx.com/bbb的方式来区分更好。

#以https为例
server {
        listen       443 ssl;
        server_name  xxx.com;

        ssl_certificate      ../cert/https.crt; #证书所在目录
        ssl_certificate_key  ../cert/https.key;

        ssl_session_timeout  5m;

        location / {
            root   html;
            index  index.html index.htm;
        }

        location /aaa/ {
            root   projects/aaa;
            index  index.html index.htm;
        }

        location /bbb/ {
            root   projects/bbb;
            index  index.html index.htm;
        }
}

也可以代理转发，自己的项目的话可以通过主机+端口号进行转发。

server {
        listen       80;
        server_name  xxx.com; #代理域名

        location / {
            root   html;
            index  index.html index.htm;
        }

        location /baidu/ {
            proxy_pass https://www.baidu.com/; #转发域名
        }

        location /xinlang/ {
            proxy_pass https://www.sina.com.cn/;
        }
}

# 总结

    通过目录方式部署的话相对麻烦一些，但是看起来比较舒服一些，如果项目不是特别多的话，这种方式还是很不错的，但如果项目比较多，xxx.com做博客或官网的话，很容易出现访问xxx.com/aaa变为访问目录，子域名的话相对简单，但要添加解析。具体怎么配置还是看个人喜好了。
