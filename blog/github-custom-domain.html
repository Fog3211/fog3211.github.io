为Github Pages自定义域名开启https

2019-02-26 09:29:10

本文记录了在自定义域名下如何开启对Github page的https支持。

# 介绍

    用Github page搭建博客已经成为了现在多数人的选择，而且一般我们都会使用自定义域名进行访问，虽然 Github已经支持了 username.github.io 强制 https 链接，在username.github.io仓库的设置中可以打开。但是如果我们开启了自定义域名，就无法让博客通过 https 来访问了。

TIP

本文以username代替github名字，请自行替换即可。

# 正文

# github page开启https

建立username.github.io仓库，开启github page服务什么的就不说了。直接说重点，首先在username.github.io仓库的设置中找到GitHub Pages，勾选强制https。

现在即可通过https://username.github.io访问博客了。

# 自定义域名

在购买域名的服务商出解析域名，将username.com(自己的域名)解析到GitHub Pages的IP地址


1.将域名的A记录解析到如下地址（一到两个即可）

185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153

2.添加CNAME记录到username.github.io

如下所示

在username.github.io仓库下新建CNAME文件并写入username.com(对应自己的域名)。

如此即可将Github Pages绑定绑定到自己的域名上了。通过username.com即可访问到username.github.io上的内容。

# 自定义域名https

http升级到https只需要在域名服务商出申请ssl证书即可。

# Github的https预览

经过以上步骤即可正常访问自己的博客了，但是你会发现原本通过Github Pages预览的项目会报https不安全。

WARNING

需要注意的是，如果开启了自定义域名支持，GitHub 提供的子域名username.github.io的https就无法生效了。 具体请移步官方Guide。

这时如果我们预览username.github.io/repo上的项目时，会自动跳转到https://username.com/repo，因为username.github.io证书失效，这时浏览器就会报安全问题。

# 解决方案

这时我们可以利用Cloudflare解决该问题。到官网注册后，填写自己的网站域名，并按提示更改DNS服务器。而后它会自动检测你的站点。

切换到 Crypto 这个标签页。将 SSL 的模式切换到 「full」。

申请ssl证书

切换到Page Rules标签页进行http=>https的跳转

这时我们再去预览username.github.io/repo上的项目就不会报错了。
