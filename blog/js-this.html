Javascript中的this指向问题

2019-06-25 20:23:22

本文将总结js中的this指向问题。

# 介绍

    在日常开发中，我们经常会用到this，而在js中这个this到底指向哪，今天我们就来聊一聊。

# 正文

    一般来说this指向的就是调用它的对象，基本有一下几种情况。

# 构造器

// this指向新创建的对象
function func(){
    console.log(this);
}

var obj = new func(); //this--->new出的新obj

# 函数名调用

// this指向window
function func(){
    console.log(this);
}

func(); // this--->window

# 对象.函数名()调用

// this指向调用它的对象
function func(){
    console.log(this);
}

var obj = {
    fun :func
};
obj.func(); // this--->obj

# apply/call绑定this

// call方式绑定this,apply同理
function add(c, d){
  return this.a + this.b + c + d;
}

var obj = {a:1, b:3};

add.call(obj, 5, 7); // 1 + 3 + 5 + 7 = 16

add.apply(obj, [10, 20]); // 1 + 3 + 10 + 20 = 34

# bind绑定this

function fun(){
  return this.a;
}

var g = fun.bind({a:"hello"});
console.log(g()); // hello

# ()=>{}箭头函数

对箭头函数而言，this指向的是它所继承的对象。

var a=0;
var obj={
    a:1,
    fun:()=>{
        console.log(this.a);
    }
}
obj.fun();//输出0

至于输出结果为什么是0而不是1，原因如下：
    首先，this是继承自父执行上下文中的this，这里的箭头函数与fun()平级（不然就与普通函数一样了），也就是箭头函数本身所在的对象为obj，但它自身的this需要向上寻找，即window，因此这里的this.a实际上表示的是window.a，所以输出的是0。

# 总结

    对于js中的this问题，虽然我也看了不少相关文章，但是关于this的水还是很深，要想吃透this还是需要继续下一番功夫啊。
