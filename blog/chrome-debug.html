论chrome控制台的正确使用姿势

2019-07-12 14:01:26

本文将介绍chrome浏览器的控制台调试方法。

# 介绍

    使用谷歌浏览器很长时间了，说来惭愧，一直以来对控制台里的一些功能还不算太了解，今天就来记录学习一下chrome的调试技巧。

# 正文

本文以chrome75为例，其他版本基本类似。

# 图标按钮

# 箭头按钮

    第一个图标：点击箭头按钮，可以通过再次点击屏幕上的点获取dom元素代码（效果同右键+检查元素）。

# 模拟设备

    第二个图标：点击设备按钮，可以切换为移动端设备（支持ipad等设备）。从左到右依次可以选择设备型号，自定义宽高，缩放倍数以及是否横屏，在移动端开发调试非常好用。

# Elements

    可以直观的查看页面元素的代码（上图）以及盒模型（如下）

    另外，双击代码可以进行修改，然后ctrl + s保存可见效果改变，但修改只是临时的，F5刷新会恢复原样。

# Console

浏览器控制台，可以直接执行js代码（换行即执行）

# Sources

# page

    网页资源页面，里面可以找到网页上所有的图片资源以及js文件，对于一些js代码是压缩过的，为了变成我们想要的可读模式，可以点击左下角的{}，然后就自动格式化了。

格式化后：

# Sinppets

    前面提到的console控制台虽然可以执行js代码，但是一旦换行就会执行，这使得我们用起来很不方便，而利用Sinppets就可以解决这一点（有点类似内置的编辑器）。
在这里面，即使换行也不会执行（右下角运行或者ctrl + enter），运行后可在控制台可以查看打印结果。

# Network

    所有网络请求相关的都可以在这里查看（包括加载图片资源，发送get/post请求等）。

另外，可以对请求的资源进行过滤，比如可以单独查看xhr请求，图片请求等等。点击每一项可以查看具体的请求状态（也可以预览传输的数据，如果有的话）

右上角可以进行网速的控制，方便模拟低网速（或者无网络）状态下的请求状况。

# Performance

    点击开始按钮然后完成一些操作之后，可以查看页面的大致性能，事件监控以及消耗cpu的记录。

# Memory

    Memory也是用来进行性能分析的，与Performance相辅相成，一个具体，一个概括。

引用陈三大佬的解释。

Heap Snapshot 能够捕捉当前页面当下 JavaScript 及 DOM 节点的内存使用情况。比如我们想看看当前页面上是否有众多的 Detached DOM 存在，则可以捕捉一个快照，然后搜索 Detached DOM。 Allocation instrumentation on timeline 可以纪录开启时段中内存分配与回收的情况。比如我们有一个操作，创建了大量的对象，则时间线上就会出现一个蓝条，框选蓝条，我们就能看到具体的内存分配状况。

# Application

    用于记录网站的数据存储，如sessionStorage,localStorage,Cookies等,当然也包括网站的pwa配置。

# Security

    可以分析这个网站的安全性，查看有效的证书等

# Audits

    内置Lighthouse，用于分析页面性能（资源加载速度，页面渲染速度，seo优化等），可以选择作为PC端还是移动端，以及网络速度的控制。

# 总结

    对于chrome，自己过去了解的还是太少，以至于只停留在简单使用上，chrome是个强大的工具，要想充分利用好它，还是需要仔细研究。
