JavaScript中的定时器

2019-02-11 22:48:56

本文将介绍js中定时器的相关用法。

# 介绍

    本文将介绍javaScript中的两个定时器setTimeout和setInterval以及HTML5新增的requestAnimationFrame。

# setTimeout

setTimeout()方法用来指定某个函数或字符串在指定的毫秒数之后执行，与之对应的是clearTimeout()，用于取消该定时器。 该函数的返回值是一个整数，表示定时器的编号。

TIP

这里的编号并不是以函数执行顺序排序的，而是按照函数声明的顺序排列的，间隔为1。


var Timer1 = setTimeout(function () {
    console.log('Timer1 :' + Timer1);
}, 2000);

var Timer2 = setTimeout(function () {
    console.log('Timer2 :' + Timer2);
}, 1000);

如下所示：

setTimeout()的第一个参数是要执行的函数，第二个函数是经过多少时间后开始执行（不写的话默认为0）。

# setInterval

    这个函数和setTimeout类似，稍稍的区别就是第二个参数代表的是时间间隔，即无限循环执行。可以用clearInterval()取消。 setTimeout可以看做是执行了一个时间周期内的setInterval()+clearInterval()。

//两种方式的结果一致
var Timer1 = setTimeout(function () {
    console.log("123");
}, 1000);

var Timer2 = setInterval(function () {
    console.log("123");
    clearInterval(Timer2);
}, 1000);

WARNING

HTML5标准规定，setTimeout的最短时间间隔是4毫秒；setInterval的最短间隔时间是10毫秒，也就是说，小于10毫秒的时间间隔会被调整到10毫秒。

# requestAnimationFrame

引用一下官方介绍：

TIP

window.requestAnimationFrame() 告诉浏览器——你希望执行一个动画，并且要求浏览器在下次重绘之前调用指定的回调函数更新动画。该方法需要传入一个回调函数作为参数，该回调函数会在浏览器下一次重绘之前执行。

    说白了就是根据系统时间间隔进行绘制，每次重绘时自动调用，不需要再自己设置时间间隔了。这样相比前两种定时器就有很大优点了：

1.CPU节能 ：当页面处于不可见或不可用状态时（最小化等等），该页面的屏幕刷新任务也会被系统暂停，requestAnimationFrame也会停止渲染，当页面被激活时开始执行。这当然就意味着更少的CPU、GPU和内存使用量。

2.函数节流 ：如果把setTimeout()的时间设为10ms，而浏览器刷新一帧的时间是16.67ms，那如果将时间设置过小就没有意义，requestAnimationFrame可以避免这种浪费性能的情况发生。

3.web动画 ：使得动画更加流畅，同时更方便调试。

    取消方法：cancelAnimationFrame()

看一下浏览器兼容还是很不错的
对于现代浏览器基本只需要IE9-即可

// 简单兼容
if (!window.requestAnimationFrame) {
   var requestAnimationFrame = function(fn) {
       var id= setTimeout(fn, 17);
        return id;
    };  
}
if (!window.cancelAnimationFrame) {
   var cancelAnimationFrame = function(id) {
        clearTimeout(id);
    };
}

// 严格兼容
if(!window.requestAnimationFrame){
    var lastTime = 0;
    var requestAnimationFrame = function(callback){
        var currTime = new Date().getTime();
        var timeToCall = Math.max(0,16.7-(currTime - lastTime));
        var id  = window.setTimeout(function(){
            callback(currTime + timeToCall);
        },timeToCall);
        lastTime = currTime + timeToCall;
        return id;
    }
}
if (!window.cancelAnimationFrame) {
   var cancelAnimationFrame = function(id) {
        clearTimeout(id);
    };
}

其他的具体可以参考官方文档。

# 总结

    对于三种定时器，setTimeout和setInterval较为相似，用于比较简单的定时，requestAnimationFrame主要用于要求比较高的动画中（比如canvas绘制粒子时），但相较前两者兼容稍差。实际应用中还是应该搭配使用比较好。
