js封装移动端长按事件函数

2019-03-12 20:03:56

本文将介绍如何用js封装一个移动端长按事件函数。

# 介绍

在开发移动端项目的时候，我们经常会用到长按事件，比如长按删除功能。但是js并没有长按事件的函数，这时就需要自己封装一个长按函数了。

# 思路

利用移动端的touchstart，touchstart事件来判断手指按在屏幕上的时间，如果超过一定时间，则认为已经触发长按事件。当然最好加上touchmove事件来判断手指是否已经移动，如果移动，则可能不是长按事件。

WARNING

本文所有代码只对移动端有效，PC端是没有长按事件的。运行效果请用Chrome手机模式查看。

# 正文

# 原生JavaScript

function longPress(obj, fn) {
    var Timer=null;
    obj.addEventListener("touchstart", function (e) {
        clearTimeout(Timer);
        Timer = setTimeout(fn, 800);
    });

    obj.addEventListener("touchmove", function (e) {
        clearTimeout(Timer);
    });

    obj.addEventListener("touchend", function (e) {
        clearTimeout(Timer);
    });
}
var obj = document.getElementById("app");
longPress(obj, () => {
    console.log("你长按了");
});

# jQuery

$.fn.longPress = function (fn) {
    var Timer = null;
    var $this = this;
    for (var i = 0; i < $this.length; i++) {
        $this[i].addEventListener("touchstart", function (e) {
            clearTimeout(Timer);
            Timer = setTimeout(fn, 800);
        });

        $this[i].addEventListener("touchmove", function (e) {
            clearTimeout(Timer);
        });

        $this[i].addEventListener("touchend", function (e) {
            clearTimeout(Timer);
        });
    }
}
$("#app").longPress(function () {
    console.log("你长按了");
});

# Vue

<template>
  <div id="app" @touchstart="touchStart" @touchmove="touchMove" @touchend="touchEnd"></div>
</template>
<script>
  export default {
    data() {
      return {
        Timer: null

WARNING

注意 `Timer = setTimeout(fn, 800);` fn后面不能加()，否则会立即执行，如果需要传递参数，应改为

Timer = setTimeout(function (e) {
        fn(item,index)；
}, 800);

关于这个问题我特意去查了一下，找到一个比较满意的解释

function Fun() {
    return 5
}
var a = Fun;
var b = Fun();

首先定义了一个简单的函数，函数是一种叫做function引用类型的实例，因此函数是一个对象。对象是保存在内存中的，函数名Fun则是指向这个对象的指针。var a = Fun，这相当于把函数名这个指针拷贝一份给变量a，函数本身并没有被复制。但是如果在Fun后面加上括号，就相当于立即执行了，也就是b会直接获取到函数的返回值。然后上面代码的问题是：事件触发是要有用户进行某项操作才会通过事件监听器调用这个操作对应的某个函数。addEventListener()方法接受三个参数，其中第二个参数就是当事件被触发时应该执行的函数，如果没有事件被触发，那函数肯定不会被调用。如果加上括号，相当于不需要监听前面的事件就可以触发。

引用自JavaScript函数后面加不加括号的区别。
