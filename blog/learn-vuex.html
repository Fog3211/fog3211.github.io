再探vuex的状态管理——Mutation与Action

2019-10-31 17:54:37

本文将详细介绍vuex中的Mutation和Action。

# 介绍

    说到状态管理，那就不能不提vuex，而vuex中的对state的操作——Mutation与Action也经常被作为面试中的考点，今天就来总结一下。

# 正文

# state

    首先，state作为vuex中的'data',用于状态的存储，相当于数据库，并且是响应式的。

// 创建
import Vue from "vue";
import Vuex from "vuex";

Vue.use(Vuex)

export default new Vuex.Store({
  state: {
    mystate:'test1'
  }
})


// 取值
this.$store.state.mystate

// 或者
import { mapState } from 'vuex'

computed:{
  ...mapState([
  'mystate'
 ])
}

// 更改状态（不推荐这种直接更改的方法）
this.$store.state.mystate='test2'

WARNING

state为什么不能直接更改的原因：
    直接更改state是异步操作，如果在更改state时还未执行完毕，其他地方有触发了该状态的更改，此时程序就会出问题。因此state的更改操作必须是同步的——即通过mutations的方式去更改state

# mutations

    前面也提到了对state而言，修改的'唯一'方式就是通过mutations。

// vuex
const store = new Vuex.Store({
  state: {
    count: 1
  },
  mutations: {
    changesomething (state,n) {
      // 变更状态
      state.count+=n
    }
  }
})
// 更改操作（这里的第二个参数是可选的）
store.commit('changesomething',n)

// 当然也可以写成对象的形式

store.commit({
  type: 'changesomething',
  amount: 10
})

// 简写
import { mapMutations } from 'vuex'
  methods: {
    ...mapMutations({
          add: 'changesomething' // 将 `this.add()` 映射为 `this.$store.commit('changesomething')`
        })
  }

# action

    上面提到了state的修改只能是同步操作，那如果我们就是想要异步操作呢？这时我们就可以用到action了。
对action而言：

Action 提交的是 mutation，而不是直接变更状态。
Action 可以包含任意异步操作。

const store = new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    changesomething (state,n) {
      state.count+=n
    }
  },
  actions: {
    changesomething ({ commit }) {
      commit('changesomething', n)
    }
  }
})

// 分发action
// 以载荷形式分发
store.dispatch('changesomething', {
  amount: 10
})

// 以对象形式分发
store.dispatch({
  type: 'changesomething',
  amount: 10
})

// 简化操作
import { mapActions } from 'vuex'
 ...mapActions({
      add: 'changesomething' // 将 `this.add()` 映射为 `this.$store.dispatch('changesomething')`
    })


// 官网的例子：可以对state进行异步操作
actions: {
  async actionA ({ commit }) {
    commit('gotData', await getData())
  },
  async actionB ({ dispatch, commit }) {
    await dispatch('actionA') // 等待 actionA 完成
    commit('gotOtherData', await getOtherData())
  }
}

# 总结

    对于Mutation与Action，简单讲就是同步与异步的关系，state只能由Mutation改变，而Action又只能操作Mutation，因而异步操作就只能作用在Action里面对Mutation的操作上了。（dispatch异步操作，commit同步操作）。
