前端瀑布流布局方式总结

2019-03-21 22:10:09

本文将总结前端常用的瀑布流布局方式。

# 介绍

    在进行图片展示的时候，我们经常会用到瀑布流布局的效果，本文就来总结一下常用的几种方式。

# 什么是瀑布流


    形如这样，视觉表现为参差不齐的多栏布局，随着页面滚动条向下滚动即为瀑布流。瀑布流布局加上懒加载，可以极大的增加用户浏览图片时的体验。

# 纯CSS方法

# Multi-columns

这是个css3新增的属性，具体文档，可以利用该方法进行瀑布流的布局。

/* 添加部分属性 */
.waterfall-container {
    /*分几列*/
    column-count: 3;
    /* 列间距 */
    column-gap: 10px;
}

这样就可以完成瀑布流的基本布局了，但是细心观察会发现有点小"bug"，该瀑布流多出来的部分会折叠到下一行的头部，这时我们可以添加break-inside属性。

/* 这里使每一列不折叠 */
.waterfall-item {
    break-inside: avoid;
}

如此就借助Multi-columns属性完成了瀑布流的布局。

# 总结

# Flexbox

    通过Flexbox也可以实现纯CSS的瀑布流布局，但是html的结构要做很大改变，要想实现几列的布局就要多建立几个div容器，个人认为这种方式不够灵活，因此没有介绍，有兴趣的话可以参考大漠老师的纯CSS实现瀑布流布局。

# Grid

    对于Grid布局，虽然它十分强大，但对于瀑布流的处理还是太过于负杂，需要指定每一个格子的高度及位置，比Flexbox还要麻烦，可以参考rachelandrew的 CSS Grid. One layout method not the only layout method。

# 结论

    总的来看，使用css进行瀑布流布局的技术还是不太成熟，个人认为如果不是有特殊的需求的话，使用js插件制作瀑布流是比较好的，毕竟简单方便，而且功能上要更加强大，既然如此，何乐而不为呢 └(^o^)┘ 。
