React项目搭建踩坑总结

2019-03-22 11:15:16

本文用来记录入门React项目开发时的踩坑集合。

# 介绍

    最近在学习react项目的开发，刚开始搭建项目结构时遇到了不少问题，特此记录一下。

# 正文

# 弹出webpack配置

# 借助CRA进行搭建
create-react-app react-demo
# 安装依赖
cd react-demo
yarn

    借助create-react-app开始快速的搭建项目的结构，但是我们往往想要自己进行webpack的配置，在vue中一般都是添加vue.config.js（vue-cli3.0），但是react好像并不能这样做，这时我们可以利用eject进行配置的弹出然后进行配置。

在package.json中可以看到eject的命令，使用node运行即可

# 进行反编译
yarn run eject

一般运行此命令是可以的，但总有不一般的时候，如下所示： 这里报错的原因是在初始化项目之后，该项目并没有本地 git 仓库，而此项目目录下又有.gitignore文件，所以此时会向上级寻找未提交的项目。

解决方法：初始化git

git init
git add .
git commit -m '初始化项目'

这样再去反编译就可以了,完成后多出来的配置目录：

# package.json依赖

细心的小伙伴可能会发现package.json中只有dependencies而没有devDependencies，按常理来说应该把webpack以及各种loader放在dev里啊。具体原因我也查了一下，引用一下kfll的回答。

在这个解决方案里面，事情是这样的：

  1. 首先要理解 react 和 webpack 分到两组去没有必要（两者均只在开发及构建阶段用到
  2. 分到哪组去只是选择问题，问题都不是很大，以我经验来看，deps 是比较适合的：
    i. 感情上：没有运行阶段，构建是一等公民
    ii. 减少约定：由于社区滥用 NODE_ENV，很可能构建脚本一开始或执行之前就设置了 NODE_ENV=production，默认情况下 devDeps 会被忽略，全都放 > deps 能避免出错。
  3. 真要分，建议构建依赖放 deps，测试依赖放 devDeps

自eject 1.0.8以后，脚手架开发人员就合并了两个依赖。总的来说就是因为放在dependencies里更保险也简单，因此做了一些不太符合规范的修改。

# webpack别名配置

用惯了vue默认配置的别名@后，在开发react也想偷个懒，那么怎么配置呢？

// 注意有两个点，这里和vue的配置不同
function resolve(dir) {
  return path.join(__dirname, '..', dir);
}
// modules中添加
alias: {
    'react-native': 'react-native-web',//这是项目自带的
    '@': resolve('src'),
},

至于为什么react的配置与vue不同呢，按道理来说应该都只是webpack相关啊。去so上搜了一下大概说是因为脚手架开发人员增加的特殊限制。。。具体请移步The create-react-app imports restriction outside of src directory

# src路径

// 在src中直接使用图片路径无法显示图片（不会报错），不论相对路径还是绝对路径
<img src="@/assets/logo.svg" className="logo" alt="logo" />

// 正确写法
import logo from '@/assets/logo.svg';
<img src={logo} className="logo" alt="logo" />

# 打包优化——map

在打包时默认配置会生成map文件（用于定位bug），在上线产品中是没什么用处的，因此可以去掉.map文件优化打包速度。

// 在webpack.config.js中搜索
// 更改shouldUseSourceMap为false

// const shouldUseSourceMap = process.env.GENERATE_SOURCEMAP !== 'false';

# redux视图更新

在使用redux的时候，在reducer中不能直接对state进行改变，因为它是引用类型，要进行对象的拷贝之后再进行赋值操作。如下

const reducer = (state = data, action) => {
    // 不能直接更改state，否则react会认为state没有改变导致视图不能更新
    let newState = Object.assign({}, state);  
    // newState的相关操作 ···
    switch (action.type) {
        case 'test':
            return newState;
        default:
            return state;
    }
}
export default reducer
