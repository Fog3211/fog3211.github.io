React中使用Symbol格式图标

2019-04-20 23:06:11

本文用来记录React使用Symbol格式图标踩坑历程。

# 介绍

    最近在开发React项目时，因为要使用彩色的图标，而在iconfont图标库中，Symbol格式是最好用的（不需要自己配置颜色）。但是按照官网上的指导进行iconfont.js文件的引入时却发生了错误。下面是踩坑记录。

# 正文

    项目是用create-react-app搭建的，因为之前使用过font-class格式的图标，本以为使用Symbol格式图标也是相似的，就直接按官方文档指示照做了，然后就报错了。

# iconfont.js引入

// 直接引入js文件
import "@/assets/iconfont/iconfont";

之后看到报错信息如下：

一开始我是完全看不懂这报错信息是什么意思的，翻译了之后也是十分晦涩难懂（应为赋值或函数调用，但却看到表达式没有未使用的表达式是什么鬼）？
后来想着去stackoverflow上问一下的，然而折腾了一段时间后发现自己声望不够（发图片需要10声望），没办法只能自己来了。。。

格式化了代码仔细看了一下报错信息，基本都是关于立即执行函数的，后来又搜了一下no-unused-expressions,终于找到了罪归祸首——eslint，没错，就是那个万恶的代码检测。

解决办法：在package.json中添加

// 忽略no-unused-expressions的检查
  "eslintConfig": {
    "extends": "react-app",
    "rules": {
      "no-unused-expressions":"off"
    }
  },

TIP

我的项目是反编译后的，即需要执行yarn eject

之后重新运行项目即可
虽然不会报错了，但是警告也是十分烦人的，根据提示在iconfont.js首行添加如下代码之后即可正常运行

// eslint-disable-next-line

# Symbol图标使用

直接按照文档来使用图标

<svg class="icon" aria-hidden="true">
  <use xlink:href="#icon-xxx"></use>
</svg>

毫无疑问又有新问题了

Namespace tags are not supported by default. React's JSX doesn't support namespace tags. You can turn on the 'throwIfNamespace' flag to bypass this warning.

解决方案有两个

  1. 这个是stackoverflow上比较推荐的
<!-- 直接改为如下代码 -->
<svg class="icon" aria-hidden="true">
  <use xlinkHref="#icon-xxx"></use>
</svg>
  1. 另一种
const useTag = `<use xlink:href=${'#icon-' + type} />`;
<svg className="icon" dangerouslySetInnerHTML={{__html: useTag }} />

# 总结

    通过这次的踩坑历程，我终于明白了什么叫Read-Search-Ask，原来通过自己分析然后解决问题真的是比直接得来答案受益地多，也是这次踩坑让自己明白不能一直按照惯性思维做事，多变通才是最重要的。
