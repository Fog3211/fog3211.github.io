vue视图无法实时更新踩坑记

2019-03-11 15:48:23

本文记录了vue项目开发中更改数组数据，视图不能实时更新的解决方案。

# 介绍

    最近在开发vue项目的时候遇到了个问题，就是在vue中更新数组数据的时候，页面视图并没有实时更新。这就不符合vue双向数据绑定的逻辑了，于是在各种踩坑后终于找到了原因，而官方文档也早已给出了解释。

TIP

由于 JavaScript 的限制，Vue 不能检测以下变动的数组：

  1. 当你利用索引直接设置一个项时，例如：vm.items[indexOfItem] = newValue
  2. 当你修改数组的长度时，例如：vm.items.length = newLength

下面我将介绍几种常见的解决方案。

# 准备工作

基本代码如下：

<template>
  <div>
    <ul>
      <li v-for="(item, index) in list">{{ item }}</li>
    </ul>
    <button @click="changeList">改变数组</button>
  </div>
</template>

<script>
export default {
  data() {
    return {
      list: ["1", "2", "3", "4"]
    };
  },
  methods: {
    changeList() {
      console.log("改变前:" + this.list);
      for (let i = 0; i < this.list.length; i++) {
        this.list[i]++;
      }
      console.log("改变后:" + this.list);
    }
  }
};
</script>

<style scoped lang="scss">
ul {
  width: 100%;
}
li {
  list-style: none;
  font-size: 30px;
  width: 200px;
  border: 1px solid #ccc;
  margin: 30px auto;
}
</style>

此时点击按钮，可以在控制台发现数组数据是已经更新了的，但是vue的视图却没有更新。

# 解决方案

# 强制刷新

使用这种方法会强制刷新页面，如果页面元素过多，则会产生空白页，用户体验不好（即使有加载动画也一样），因此不推荐。

location.reload();
// 或者
this.$router.go(0);

# provide/inject

先给出官方介绍地址。官方文档。

这种方法需要在App.vue页面做一下修改

export default {
  name: "App",
  provide: function() {
    return {
      reload: this.reload
    };
  },
  data: function() {
    return {
      isRouterAlive: true
    };
  },
  methods: {
    reload() {
      this.isRouterAlive = false;
      this.$nextTick(function() {
        this.isRouterAlive = true;
      });
    }
  }
};

然后在当前vue页面注入reload方法

export default {
 inject:['reload'],
   methods: {
    changeList() {
      console.log("改变前:" + this.list);
      for (let i = 0; i < this.list.length; i++) {
        this.list[i]++;
      }
      console.log("改变后:" + this.list);
      this.reload();//函数执行完后调用该方法
    }
  }
}

这是官方文档的说明，可见正式开发中此方法的优先级并不高。

provide 和 inject 主要为高阶插件/组件库提供用例。并不推荐直接用于应用程序代码中。

# 数组更新检测（推荐）

官方文档

这种方法是我用过最好的了，即Vue.set(arr, index, value)

// 首先需要在当前页面引入vue
import Vue from "vue";
// 使用Vue.set();进行数组数据的更改
Vue.set(arr, index, value);

// 对于本例,改为如下函数即可
changeList() {
  console.log("改变前:" + this.list);
  for (let i = 0; i < this.list.length; i++) {
    Vue.set(this.list, i, ++this.list[i]);
  }
  console.log("改变后:" + this.list);
}

或者使用arr.splice(index, 1, value)

// 使用splice();进行数组数据的更改
arr.splice(index, 1, value);

// 对于本例,改为如下函数即可
changeList() {
  console.log("改变前:" + this.list);
  for (let i = 0; i < this.list.length; i++) {
    this.list.splice(i, 1, ++this.list[i])
  }
  console.log("改变后:" + this.list);
}

对于以上两种方式在一维数组中没有什么显著的区别，但是如果对于二维数组，前者就无法完成视图的更新了，必须采用如下方式。

// 可以实现arr[index][jndex]=value;的效果
changeArrValue(arr, index, jndex, value) {
      let array = arr[index];
      array[jndex] = value;
      arr.splice(index, 1, array);
    }

# 总结

    对于vue数组视图更新不及时的"bug",个人认为官方给出的Vue.set()和splice()方法都是比较好的，可以解决大多数情况的问题。虽然在多维数组时可能会稍微麻烦一点，但一般也用不到二维以上的数组，因此对于数组问题这样处理应该是最好的办法了。
