使用ssh为Git配置多用户

2019-09-23 10:30:20

本文将介绍如何使用ssh为git配置多个用户。

# 介绍

    Git作为一个版本控制工具，极大的提高了多人合作开发的效率，但是当我们进入公司以后，就需要使用新的git账号了（一般是在内网搭建的git社区），而这时我们又不想放弃自己本来的Git账户，那该怎么办呢？这时我们可以通过配置多个账户来解决问题。

# 正文

# 准备条件

假设我们需要3个git账号（github.com，gitlab.com，以及公司的社区版，这里假设以192.168.0.0为例）

# 创建公钥和私钥

# 输入自己的邮箱
ssh-keygen -t rsa -C "fog3211@163.com"

这时会看到:

然后提示我们为文件命名（回车的话是默认名字，对应一个账号的情况），对于多个账可以更具自己的爱好去命名。

Enter file in which to save the key (/c/Users/fog3211/.ssh/id_rsa): /c/Users/fog3211/.ssh/id_rsa_github

WARNING

  1. 这里需要写全路径，不然会在当前目录下创建文件
  2. .ssh目录默认是隐藏的，需要选择显示隐藏文件夹。

然后就是输入密码阶段了（回车默认没有密码）

以上便创建好了公钥和私钥。

# 设置公钥

这个很简单，登录网页版github或gitlab，在设置里找到SSH Keys，新建并粘贴刚才创建好的id_rsa.pub文件（用记事本打开即可）。

# 管理公钥

在公钥的目录（/c/Users/fog3211/.ssh/）下面新建一个config文件(没有后缀名)，然后自己进行配置

# 配置文件参数
# Host : Host可以看作是一个你要识别的模式，对识别的模式，进行配置对应的的主机名和ssh文件
# HostName : 要登录主机的主机名
# User : 登录名
# IdentityFile : 指明上面User对应的identityFile路径

# gitlab
Host gitlab.com
    HostName gitlab.com
    User fog3211 
    PreferredAuthentications publickey
    IdentityFile ~/.ssh/id_rsa_gitlab
# github
Host github.com
    HostName github.com
    User lisi
    PreferredAuthentications publickey
    IdentityFile ~/.ssh/id_rsa_github
# local
Host 192.168.0.0
    HostName 192.168.0.0
    User zhangsan
    PreferredAuthentications publickey
    IdentityFile ~/.ssh/pica_gitlab

    这里的配置文件会根据你访问的域名进行判断去使用哪一个私钥，而用户名的话一般是用于区分昵称和真名（公司的话一般都是真名拼音，自己用的话一般就是昵称的吧）

# 添加及验证

    这时一般就ok了，但是ssh会有缓存，因此如果我们不是第一次使用git，那就需要清一下缓存。

ssh-add -D

如果有报错的话，提示Could not open a connection to your authentication agent.

输入以下命令并重复上一步即可

eval `ssh-agent -s`

然后重新添加

# 对应自己的文件名
ssh-add ~/.ssh/id_rsa

#  查看添加结果 
ssh-add -l

# 总结

    之前都是自己使用一个git账号进行开发，自从进了公司之后才发现原来的很多习惯和配置都需要调整，像git配置就是这样，单个账号用习惯了再次添加的话会默认覆盖第一个，不过如果不是这样恐怕我也不会知道怎么去配置多用户吧。。。
