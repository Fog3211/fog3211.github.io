javaScript作用域问题

2019-07-29 20:26:11

本文将介绍javaScript中的作用域问题。

# 介绍

    首先，什么是作用域？作用域是一个能够表示变量或函数影响范围的东西，在作用域外变量或函数是“不存在”的。

# 正文

# 变量作用域

    作用域的权限从外到内依次递增，里面的部分可以访问外面部分的属性，而外面的部分无法访问里面部分的属性;如B可以访问全局的d属性，而window无法访问除d以外的属性；同级别的B和C也只能访问到自己作用域的属性，如B无法访问c属性，C访问的属性也只是自己的C.c

# 函数作用域

# 提升

1.对于函数声明，函数声明会连通命名和函数体一起被提升至作用域顶部

function test() {
    foo();

    function foo() {
        alert("打印");
    }
}

test();

2.函数表达式只有命名会被提升，定义的函数体则不会，因此直接访问会报错

function test() {
    foo();

    var foo = function () {
        alert("不会打印");
    }
}

test();

# 立即执行函数

立即执行函数 IIFE：所声明的变量仅保存在函数体内的局部作用域内。

var a = 2;

(function IIFE() {
    var a = 3;
    console.log(a); //3
})();

console.log(a); //2

作用 ：

1.令函数中声明的变量绕过变量声明前置规则
2.避免新变量被解释成全局变量或函数名占用全局变量名的情况
3.在禁止访问函数内变量声明的情况下，允许外部对函数的调用

# 静态作用域 & 动态作用域

var value = 1;

function foo() {
    console.log(value);
}

function bar() {
    var value = 2;
    foo();
}

bar();

静态作用域（词法作用域，JS，C, C++, Python, Java等大多数语言）：函数的作用域基于函数创建的位置，打印结果为1

分析：foo()执行的时候，自己本身里面没有value，然后向上寻找value，函数foo创建的位置在全局中，因此打印的value为1；

动态作用域（Emacs Lisp, Common Lisp，Perl）：打印结果为2

分析：执行 foo 函数，foo 函数没有找到 value，向上寻找调用函数的作用域，也就是 bar 函数内部查找 value 变量，即打印value为2

# 块级作用域 let & const

1.不使用关键字声明的变量默认属于全局作用域,如 a=0；在任何地方都可以访问到a的值

function fun() {
    a = 1;
}

function test() {
    let b = 2;
}

fun();
test();

console.log(a); //1
console.log(a); //报错

2.var声明变量后,后面的部分可以访问到已声明的属性（不会销毁）

for (var i = 0; i < 5; i++) {
    console.log("局部变量的i: " + i); //0,1,2,3,4
}

console.log("全局变量的i: " + i); //5

3.let/const类似强类型语言，拥有块级作用域（出了自己的{}会被销毁）

for (let i = 0; i < 5; i++) {
    console.log("局部变量的i: " + i); //0,1,2,3,4
}

console.log("全局变量的i: " + i); //报错：undefined

# 欺骗作用域 eval & with

    对于js的作用域大部分情况下是会在声明的时候就已经决定好了，但通过eval和with却可以对作用域进行改变（非严格模式下，并会降低性能）

eval:eval可以执行传入的字符串，通过eval可以进行属性的“插入”，从而达到改变作用域的目的

function foo(str, a) {
    eval(str);
    console.log(a, b);
}
var b = 2;

foo("var b=3;", 1); //1,3

with:


// with执行的对a的赋值，由于对象内并没有a属性，因此o.a为未定义，而with内的a=2赋值因为没有关键字从而将全局变量的a变为了2.

function foo(obj) {
    with(obj) {
        a = 2;
    }
}
var o = {
    b: 3
};

foo();

console.log(o.a); //undefined
console.log(a); //2
