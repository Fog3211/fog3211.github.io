如何不借助脚手架工具开发Vue项目

2018-11-13 23:25:57

本文将介绍如何在不借助脚手架的情况下搭建vue开发环境。

# 介绍

Vue-cli就不多说了，vue官方提供的脚手架工具，可以快速开始零配置原型开发，但是里面有些东西用不到使webpack打包变得笨重，本文将讲述如何不借助Vue脚手架工具来搭建一个Vue项目,主要运用webpack4.x来进行环境的配置。

# 开始搭建

# 项目环境

node>=8.0
webpack4.25.1
vue2.5.17

WARNING

这里要讲一下，如果打算用webpack3.x进行搭建，npm安装的babel-loader模块版本不能高于7.1.5，否则打包js时会报错。

# 项目目录

首先npm初始化

npm init -y

安装必要的模块

npm i babel-loader babel-core babel-preset-env  webpack webpack-cli webpack-dev-server -D

搭建vue目录结构

vue-withoutcli
  ├──── dist
  ├──── src
  │      ├── compontents  
  │      ├── public  
  │      ├── router  
  │      │      └── index.js  
  │      ├── style  
  │      ├── App.vue  
  │      └── main.js
  ├──── package-lock.json
  ├──── package.json
  └──── webpack.config.js

其中src/main.js是打包入口文件，dist目录用于存放打包后的文件，webpack.config.js用来配置打包信息 compontents存放vue组件，public存放图片等资源，router路由，style存放css,less等样式文件

# 开始搭建

# 基本配置

在webpack.config.js中配置

//webpack.config.js

module.exports = {
    entry: __dirname + '/src/main.js',    //入口文件
    output: {
        path: __dirname + '/dist',    //打包存放目录
        filename: 'bundle.js'
    },
    module: {},
    resolve: {},
    plugins: [],
    devServer: {
        contentBase: './dist',
       stats: {
          colors: true
        },
        historyApiFallback: true,
        inline: true,
       open:true,
    }
}

package.json添加

{
"scripts": {
    "dev": "webpack-dev-server",
    "build": "webpack"
  }
}

# 初次打包

此时运行npm run build命令即可进行打包,目录如下

为解决打包后没有生成html文件,这里需要借助webpack的html-webpack-plugin插件,使用npm安装html-webpack-plugin模块，并在项目根目录下新建index.html文件

<!-- index.html -->
<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width,initial-scale=1.0">
    <title>vue-withoutcli</title>
  </head>
  <body>
    <div id="app"></div>
    <!-- built files will be auto injected -->
  </body>
</html>

//webpack.config.js添加
const HtmlWebpackPlugin = require('html-webpack-plugin')
module.exports = {
   plugins: [
        new HtmlWebpackPlugin({
            filename: 'index.html',
            template: 'index.html',
            inject: true,
        })
    ],
}

# Vue雏形

# package.json
npm i vue vue-loader vue-template-compiler vue-router -D

入口文件引入vue,创建vue实例并挂载到#app上

//main.js
import Vue from 'vue'
import App from './App.vue'
import router from './router'
new Vue({
  el: '#app',
  router: router,
  template: '<App/>',
  components: { App }
})

主组件

安装用于对css样式进行转换的loader

# css
npm i css-loader  style-loader -D

当然webpack也要添加一些配置，

// webpack.config.js

const path = require('path')

module.exports = {
  module: {
    loaders: [{
        test: /\.js$/,
        exclude: /node_modules/,
        loader: 'babel-loader',
        query: {
          presets: ['env']
        }
      },
      {
        test: /\.vue$/,
        loader: 'vue-loader'
      },
    ]
  },
  resolve: {
    extensions: ['.js', '.vue', '.json'],
    alias: {
      'vue$': 'vue/dist/vue.js',
      '@': path.resolve(__dirname, "src")
    }
  },
  }

具体组件就不贴了，末尾有github地址，可以对着参考

WARNING

此处小坑，webpack的alias（路径别名）只能处理js，如@表示src目录,而在css中无法识别，~是css-loader约定标识符，以~开头的路径会以 require 的方式定位.

css预处理器，以这里less为例

npm安装依赖

# package.json
npm i less less-loader -D

webpack.config.js里添加loader配置

// webpack.config.js
module.exports = {
  module: {
    loaders: [{
      test: /\.less$/,
      loader: "style-loader!css-loader!less-loader",
    } ]
  }
}

WARNING

需要注意lang="less"不能省略，否则会被当成css文件，未经过less-loader转换会报错
