JavaScript封装函数防抖与节流

2019-02-15 13:15:02

本文将介绍如何用js封装函数防抖与节流。

# 介绍

# 解释

TIP

函数防抖（debounce）：让一个函数在一定间隔内没有被调用时，才开始执行被调用方法。

函数节流（throttle）：是让一个函数无法在很短的时间间隔内连续调用，当上一次函数执行后过了规定的时间间隔，才能进行下一次该函数的调用。

# 为什么要做防抖与节流

    在前端开发中我们可能会使用到一些被频繁触发的函数（如scroll，mousemove等），如果正常绑定的话，会在短时间内多次触发，不仅会大量浪费性能，更可能导致界面卡顿，甚至浏览器的崩溃。而函数防抖(debounce)和节流(throttle)就是为了解决类似需求应运而生的。

# 实现

# 函数防抖

   打个不太恰当的比方就像你在浏览器控制台Network加载资源，每次刷新页面，就会发起请求，但这时还未全部请求到数据，如果高频的刷新页面，只能在最后一次刷新并且等待一段请求时间后才可以完成全部请求。

简单实现如下

function debounce(fun, delay) {
    var timer = null;
    return function () {
        var that = this,
            args = arguments;
        clearTimeout(timer);
        timer = setTimeout(function () {
            fun.apply(that, args);
        }, delay);
    }
}

document.getElementsByTagName("body")[0].onmousemove = debounce(function (e) {
    var e = e || window.event;
    console.log(`鼠标当前位置：（${e.screenX}，${e.screenY}）`);
}, 1000);

复杂实现可以参考underscore.js

_.debounce = function(func, wait, immediate) {
  var timeout, result;

  var later = function(context, args) {
    timeout = null;
    if (args) result = func.apply(context, args);
  };

  var debounced = restArguments(function(args) {
    if (timeout) clearTimeout(timeout);
    if (immediate) {
      var callNow = !timeout;
      timeout = setTimeout(later, wait);
      if (callNow) result = func.apply(this, args);
    } else {
      timeout = _.delay(later, wait, this, args);
    }

    return result;
  });

  debounced.cancel = function() {
    clearTimeout(timeout);
    timeout = null;
  };

  return debounced;
};

# 函数节流

简单讲就像冰棱上的水滴只有攒到一定量的时候才开始滴落。

简单实现

function throttle(fun, delay) {
    var last = 0;
    return function () {
        var that = this,
            args = arguments;
        var now = new Date();
        if (now - last > delay) {
            fun.apply(that, args);
            last = now;
        }
    }
}

document.getElementsByTagName("body")[0].onmousemove = throttle(function (e) {
    var e = e || window.event;
    console.log(`鼠标当前位置：（${e.screenX}，${e.screenY}）`);
}, 1000);

复杂实现同参考underscore.js

_.throttle = function(func, wait, options) {
  var timeout, context, args, result;
  var previous = 0;
  if (!options) options = {};

  var later = function() {
    previous = options.leading === false ? 0 : _.now();
    timeout = null;
    result = func.apply(context, args);
    if (!timeout) context = args = null;
  };

  var throttled = function() {
    var now = _.now();
    if (!previous && options.leading === false) previous = now;
    var remaining = wait - (now - previous);
    context = this;
    args = arguments;
    if (remaining <= 0 || remaining > wait) {
      if (timeout) {
        clearTimeout(timeout);
        timeout = null;
      }
      previous = now;
      result = func.apply(context, args);
      if (!timeout) context = args = null;
    } else if (!timeout && options.trailing !== false) {
      timeout = setTimeout(later, remaining);
    }
    return result;
  };

  throttled.cancel = function() {
    clearTimeout(timeout);
    previous = 0;
    timeout = context = args = null;
  };

  return throttled;
};

# 总结

    两个方法都是用来提升前端性能，减轻浏览器压力。不同点在于函数防抖在函数高频调用时不会执行，需要等待一段时延，而函数节流在高频调用函数时依旧会执行，它会按照固定时延执行。
